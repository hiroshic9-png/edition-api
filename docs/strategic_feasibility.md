# EDITION — 戦略的実現可能性分析

> AI認証能力の構築期間、競争優位の防御可能性、事業に必要な法的許認可の分析

---

## Q1. AI認証能力の構築にどのくらいの時間がかかるのか？

### 結論: 段階的に3年。ただし「使えるレベル」は12ヶ月で到達可能。

```
Phase 0 (Month 0-3):  基盤構築 ←【今ここ】
  ├─ データ収集パイプラインの設計
  ├─ メトロポリタン美術館CC0 / ColBase CC BY 4.0 データの体系的収集
  ├─ 既存のオープンソースCNN（ResNet, ViT）のFine-tuning実験
  └─ 刀剣カテゴリのPoC（刃文パターン分類の実証実験）

Phase 1 (Month 3-12): MVP — 「参考意見」レベル
  ├─ 刃文パターン分類AI（五箇伝の判別 — 精度70-80%目標）
  ├─ 陶磁器の窯跡マッチング（主要6窯の判別）
  ├─ 浮世絵の摺り状態評価（初摺 vs 後摺の判別）
  ├─ 来歴文書のOCR + 構造化（箱書き・花押のデジタル化）
  └─ 専門家ネットワークとのフィードバックループ構築

Phase 2 (Month 12-24): 実用 — 「補助ツール」レベル
  ├─ 精度85-90%の複数カテゴリ判定
  ├─ 市場異常検知システム（同一品の複数出品、不自然な来歴等）
  ├─ ブロックチェーン来歴管理のβ版
  └─ NBTHK等の鑑定機関との比較検証

Phase 3 (Month 24-36): 権威 — 「EDITION認証」ブランド確立
  ├─ 精度95%+の統合認証システム
  ├─ 科学分析パートナーとの連携（XRF, FTIRデータの統合）
  ├─ 「EDITION Certified」の市場認知確立
  └─ 継続的学習ループの自律運転
```

### なぜ時間がかかるのか？（ボトルネック）

| ボトルネック | 詳細 | 解決策 |
|------------|------|--------|
| **データ収集** | 認証済みの高品質画像が数千点必要。美術品データは断片的で非公開が多い | Met Museum CC0（50万点）, ColBase（国立博物館）をベースに、独自撮影・パートナーシップで補完 |
| **「正解」の定義** | AIが学ぶには「本物」と「偽物」の両方が必要。偽物のデータセットは公開されていない | 既知の贋作事例の文献データ化。合成データ（GANで生成した「偽の刃文」等）でデータ拡張 |
| **専門家の関与** | 訓練データの正確性を保証するには専門家の検証が不可欠 | NBTHK認定者、陶磁器研究者等とのアドバイザリー関係を構築 |
| **ドメイン知識の構造化** | 「良い刃文とは何か」を機械が理解可能な形に落とし込む作業 | 専門書の体系的デジタル化。鑑定基準のオントロジー構築 |

### Art Recognitionの事例（参考）

チューリッヒのArt Recognition社は、**設立から商用レベルまで約3年**かかっている。ただし彼らは西洋絵画という比較的データが豊富な分野。日本文化資産はデータが少ない分、構築に時間がかかるが、逆に**参入障壁も高い**。

## Q2. 技術的に簡単に真似できるものなのか？

### 結論: 極めて難しい。EDITIONの「堀」は時間とともに深くなる。

### 真似が困難な5つの理由

#### 1. **データの非代替性**（最大の堀）
- 日本文化資産の高品質な認証済み画像データは、**地道に蓄積する以外に入手手段がない**
- サザビーズは数十年分のオークション記録と画像を蓄積している → それが武器
- EDITIONが5年間蓄積したデータを、後発が「お金を出して買う」ことはできない
- **データ量ではなく「データの質と文脈」が価値** — 誰が、いつ、どの基準で認証したかというメタデータが不可欠

#### 2. **日本語 + 専門知識の二重障壁**
- 箱書きの崩し字解読、花押の作家判定、蔵帳の文献調査 — これは英語圏のAI企業には構築不可能
- NBTHK、各窯元、刀匠との関係性は「コードを書く」だけでは構築できない
- **西洋のAI認証企業が日本市場に参入するハードル >> EDITIONが日本市場を先取りするハードル**

#### 3. **フィードバックループの複利効果**
- 認証を行うたびにデータが蓄積 → モデルが改善 → より正確な認証 → より多くの依頼 → さらにデータが蓄積
- この「フライホイール」が回り始めると、後発は追いつけない
- **先に回し始めた者が勝つ**

#### 4. **専門家ネットワークの排他性**
- 日本の古美術業界は「信頼の関係」で成り立つ
- NBTHK、主要な古美術商、博物館との関係は一朝一夕では構築できない
- 一度信頼を獲得すれば、競合が同じ関係を築くのは極めて困難

#### 5. **マルチモーダル統合の技術的複雑性**
- 画像分析だけなら真似できる。しかし：
  - 画像 + 化学分析データ + 来歴文書 + 市場データ を**統合して**判定するシステムは、各レイヤーのドメイン知識が必要
  - これは「汎用AIモデル」では解けない問題

### ただし注意すべきリスク

| リスク | 内容 |
|--------|------|
| **大手テック企業の参入** | Google/Microsoftが「文化遺産AI」を始める可能性。ただし「ニッチすぎて」彼らの優先度は低い |
| **中国の急追** | 中国企業が同様の文化資産AIを構築する可能性。ただし日本美術の専門性では差別化可能 |
| **オープンソース化** | 基礎技術（CNN, ViT）は公開されている。差別化はデータとドメイン知識 |

## Q3. 事業に必要な免許・許認可

### 結論: ハードルは低い。「古物商許可」が唯一の必須許可。

### 事業形態別の必要許認可

| 事業 | 必要な許認可 | 取得難易度 | 費用 | 期間 |
|------|-----------|----------|------|------|
| **コレクション仲介（売買）** | ★ **古物商許可** | 低 | 19,000円 | 約40日 |
| **オークション運営** | ★ **古物商許可** + **古物競りあっせん業届出** | 低 | 19,000円 | 約40日 |
| **鑑定サービス** | **不要**（国家資格なし） | — | — | — |
| **AI認証サービス** | **不要** | — | — | — |
| **エスクロー決済** | ⚠️ **資金移動業登録**（スキームによる） | 高 | 要検討 | 数ヶ月 |

### 詳細

#### 古物商許可（必須・最優先で取得すべき）
- **管轄**: 営業所所在地の都道府県公安委員会（警察署経由）
- **費用**: 申請手数料 **19,000円** + 書類発行実費 数千円
- **審査期間**: **40日**（平日）
- **個人/法人**: どちらでも申請可。法人の場合は定款・登記簿が追加で必要
- **注意**: バーチャルオフィスは原則不可。実体のある営業所が必要
- **無許可営業の罰則**: 3年以下の懲役 or 100万円以下の罰金

#### 鑑定サービス — 国家資格不要
- 日本には**美術品鑑定の国家資格は存在しない**
- 2025年10月に「美術品鑑定士ライセンス」（公益財団法人 頌古会）が民間検定として開始 → ただし法的義務なし
- **実務上の信頼性**は、実績・専門性・機関との関係で構築するもの
- EDITIONのAI認証は「鑑定」ではなく「認証支援データの提供」とポジショニングすれば、法的リスクも最小化

#### 売買仲介 — 追加で注意すべき法律

| 法律 | 関連する場面 |
|------|-----------|
| **特定商取引法** | 通信販売（EC）で美術品を販売する場合。広告表示義務、クーリングオフ |
| **消費者契約法** | B2C取引で不利な契約条項の無効化 |
| **景品表示法** | 品質・価格の誤認表示禁止。「鑑定済み」等の表記に注意 |
| **著作権法** | 作品画像の使用許諾。購入≠著作権取得 |
| **犯罪収益移転防止法** | 高額取引のマネーロンダリング防止。本人確認義務 |

#### エスクロー決済 — 慎重な設計が必要
- 他人の金銭を預かって送金する行為は「**資金移動業**」の登録が必要（資金決済法）
- 「収納代行」形式で回避するケースもあるが、実態が為替取引とみなされるリスク
- **推奨**: Phase 1では既存の決済プラットフォーム（Stripe等）を利用。独自エスクローはPhase 3以降

### 推奨アクション

```
今すぐ:
  └─ 古物商許可の申請（19,000円、40日で取得）

3ヶ月以内:
  └─ 特定商取引法に準拠したサイト表記の整備
  └─ 利用規約・プライバシーポリシーの作成

6ヶ月以内:
  └─ AML（マネーロンダリング防止）ポリシーの策定
  └─ 顧問弁護士の選定（美術品取引 or IT法務に精通した者）
```

| 質問 | 回答 |

| **時間** | 「使える」レベルまで12ヶ月。「権威」レベルまで36ヶ月。データ収集がボトルネック |
