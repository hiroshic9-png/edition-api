Price Estimation

AI-powered hedonic pricing for Japanese art. Uses a 17-feature ensemble model (HistGradientBoosting + RandomForest) trained on 49,968 auction records.

POST /estimate/price Estimate fair market value

Returns a hedonic price estimate with confidence range based on artist history, medium, size, and market context.

Request Body

ParameterTypeDescription
artiststringArtist name (Japanese or Roman)
mediumstringMedium classification (oil, acrylic, silkscreen, lithograph, etc.)
width_cmnumberWidth in centimeters
height_cmnumberHeight in centimeters
estimate_jpynumberAuction house estimate (optional, improves accuracy)

Example

curl -X POST https://api.edition.sh/estimate/price \
  -H "Content-Type: application/json" \
  -d '{
    "artist": "草間彌生",
    "medium": "acrylic",
    "width_cm": 50,
    "height_cm": 50
  }'

Response

{
  "estimate_jpy": 1723782,
  "range_low": 1342826,
  "range_high": 2104737,
  "confidence": "medium",
  "model_version": "0.9",
  "model_r2": 0.900
}
GET /estimate/artist/{name} Artist price statistics

Returns comprehensive market statistics for a specific artist, including lot count, average/median prices, and auction house distribution.

curl https://api.edition.sh/estimate/artist/草間彌生
{
  "artist": "草間　彌生",
  "lot_count": 1319,
  "avg_price": 7914157,
  "median_price": 2760000,
  "max_price": 247250000,
  "n_houses": 3,
  "houses": ["Mainichi", "SBI", "Shinwa"]
}
GET /estimate/model Model metadata

Returns current model version, R², feature count, and training statistics.

Authentication & Assessment

Forensic-grade assessment endpoints for text-based and image-based authentication analysis.

POST /assess/text Text-based authentication
ParameterTypeDescription
textstringDescription of the item to assess
categorystringOptional: swords, ceramics, paintings, etc.
POST /assess/image-upload Image-based authentication

Upload an image for AI-powered visual analysis. Multipart form data with an `image` field.

Provenance Search

Cross-house provenance chain search across 49,968 auction records and the Neo4j knowledge graph.

POST /provenance/search Search provenance chain
ParameterTypeDescription
artiststringArtist name
titlestringOptional: Work title for precise matching
limitnumberMax results (default: 20)
GET /provenance/artist/{name} Artist timeline

Returns chronological auction timeline for an artist across all houses.

Knowledge Graph

Neo4j-powered graph queries for cross-house analysis. 7,258 Artist nodes, 49,968 Lot nodes, 325,130 CROSS_HOUSE relationships.

GET /graph/stats Graph database statistics

Returns node/relationship counts and top artists by total auction value.

GET /graph/artist/{name} Artist graph profile

Returns graph-based artist profile including connected houses, lot count, and cross-house relationships.

MCP Integration

KANTEISHI is available as an MCP (Model Context Protocol) server for AI agent integration via Claude Desktop and other MCP-compatible clients.

Claude Desktop Config
{
  "mcpServers": {
    "kanteishi": {
      "command": "python",
      "args": ["kanteishi_mcp.py"]
    }
  }
}
